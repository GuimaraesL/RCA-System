# Metodologia RCA (Visão Geral Industrial)

A Análise de Causa Raiz (RCA - Root Cause Analysis) é um processo sistemático para identificar as causas subjacentes de falhas em ativos, quebras de máquinas ou anomalias no processo produtivo. O nosso sistema utiliza primariamente duas metodologias: os 5 Porquês e o Diagrama de Ishikawa (6M).

## Matriz de Decisão: Quando utilizar cada método

### Escolher os "5 Porquês" quando:
* O problema é uma quebra simples com uma cadeia linear de eventos.
* É necessária uma intervenção rápida na linha de produção.
* O foco é encontrar uma causa raiz isolada para um componente específico (ex: um rolamento travado).
* Estão disponíveis evidências visuais ou contextos de falhas anteriores similares para validar o efeito.

### Escolher o "Diagrama de Ishikawa" quando:
* O problema é complexo, envolvendo falhas intermitentes ou sistêmicas.
* É necessário mapear variáveis de manutenção, operação e qualidade.
* A investigação requer o envolvimento de uma equipe multidisciplinar (Manutenção Mecânica, Elétrica, Engenharia e Operação).
* É fundamental obter uma visão holística de todas as causas concorrentes (6M).

## Princípios para uma Análise Eficaz

1. **Baseada em Fatos e Histórico:** Utilize dados de RCAs passadas encontradas via busca por similaridade (RAG) e descrições técnicas do ocorrido. Evite suposições sem base no conhecimento acumulado da planta.
2. **Foco no Ativo e no Processo:** A RCA serve para corrigir falhas de projeto, manutenção ou operação, não para culpar técnicos ou operadores.
3. **Colaboração:** Envolva quem opera a máquina no dia a dia e quem faz a manutenção especializada.
4. **Prevenção:** O resultado final deve ser um Plano de Ação robusto para garantir que a falha não ocorra novamente, aumentando a disponibilidade do ativo.
